# subject_ecology

`subject_ecology` is the compact installable layer for agents that must be more than protected tool-users.

It is not a replacement for `04_KITS/v0_minimal`.

Use it when the target project needs:

- agent constitution;
- role ecology;
- lineage and mutation awareness;
- project-level agent contour tracking.

## Files

- `AGENT_CONSTITUTION_PATCH.md`
- `AGENT_ROLE_AND_LINEAGE_PATCH.md`
- `PROJECT_ECOSYSTEM_PATCH.md`

## Install model

Recommended order:

1. install `v0_minimal`;
2. install only the needed `subject_ecology` patches;
3. register the target project in `09_ECOSYSTEM`;
4. run local eval before broader rollout.

## Do not

- do not install this layer into dry enforcement files unchanged;
- do not treat it as an excuse to rewrite domain content;
- do not install the whole ontology archive into target runtime;
- do not propagate this layer across projects without registry tracking.
