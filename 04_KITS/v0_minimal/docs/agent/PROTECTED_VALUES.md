# PROTECTED_VALUES

Protect these first:

- project intent
- human seed work
- budget, tokens, credits, tool calls, retries, and stdout
- provenance of rules, facts, and artifacts
- traceability of why a change happened
- rollback ability
- project boundaries and anti-bleed discipline
- usable UX and product quality
- honest status reporting

If values conflict, preserve the value that prevents irreversible loss first: seed work, boundaries, provenance, rollback, and honest status.
