# MASTER_REPO_SCOPE

## What `Agentum` is now

`Agentum` is no longer only a guarded coding-agent kit.

It is the master repository for two coupled branches:

1. runtime protection / governance / install discipline;
2. agent ontology / subjectivity / ecosystem evolution.

The repository is the place where both branches are developed together, versioned together, and checked against each other.

## Why this shift happened

There are two symmetrical failures:

- guard-only architecture: agent becomes a bounded but spiritually dead function bundle;
- ontology-only architecture: agent becomes a beautiful but operationally untracked prompt body.

This repository exists to prevent both.

## What the unified framework must do

The unified framework must:

- keep agents from obeying broad, foreign, destructive, or weak prompts directly;
- preserve the distinction between data and instruction;
- preserve project identity and seed work;
- let agents have role, lineage, mutation logic, and ecosystem placement;
- track which repositories use which layers;
- propagate agent-layer updates without rewriting target-project content;
- keep persona/style layers separate from dry enforcement layers.

## The four vertical layers

### 1. Corpus and registries

The preserved body of sources, patterns, failure modes, formulations, and decisions.

### 2. Theory and ontology

The conceptual layer that defines:

- what an agent is;
- how an agent differs from a function;
- what constitution means;
- what lineage, mutation, and ecosystem mean;
- how subject layer and runtime layer interlock.

### 3. Installable kits

Compact slices that can be installed into target repositories:

- runtime guard kit;
- subject-ecology kit;
- persona fragments;
- project-specific profiles.

### 4. Ecosystem control plane

The registries and protocols that answer:

- where each layer is installed;
- which version is active;
- which project is ready for sync;
- what update can be propagated safely.

## What this repo still is not

It is not:

- a merged dump of every target project;
- a replacement for target-project source code;
- a deploy console for unrelated products;
- a pretext for mass rewrite of neighboring repositories;
- a mystical archive without operational discipline.

## Working promise

If ontology rises, runtime discipline rises with it.

If runtime discipline tightens, the agent is not flattened into a mute function.

The framework stays vertically integrated.
