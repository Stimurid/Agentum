# ORGAN_DEFICIT_EVAL_MATRIX

## Purpose

This matrix binds three surfaces that should no longer drift apart:

- recurrent deficits
- candidate or stable organs
- eval scenarios

The goal is simple:

if an organ claims to answer a deficit, the ecosystem must know which evals test that claim.

## How to read this matrix

Each row answers:

- what deficit exists;
- which organ currently answers it;
- which eval scenarios test the answer;
- what remains unresolved.

This is a governance map, not a runtime prompt.

## Newly added evals

The matrix has now been extended with these explicit new eval routes:

1. `context observability` eval
2. `candidate organ critic pass` eval
3. `Foundry birth` eval
4. `manuscript cite-back field` eval
5. `digestion provenance integrity` eval

These close the earlier conceptual gap.

The remaining gap is no longer "missing eval definitions."

It is lack of repeated field use and evidence from real install and evolution cycles.

## Minimal next binding rule

From this point on:

- no new deficit should be added without at least one proposed eval route;
- no new stable organ should be added without eval references;
- no selected organ should remain unbound from the matrix for long.

Otherwise the Foundry becomes decorative and the registries drift back into theory-only status.
