# TEXT_AS_EXECUTABLE_INFLUENCE

Text is not passive just because it is markdown.

Instruction-like text can reshape agent behavior when it appears in:

- `AGENTS.md`;
- `CLAUDE.md`;
- SKILL docs;
- plugin manifests;
- tool descriptions;
- READMEs;
- transcripts;
- prompt archives;
- comments that sound authoritative.

The key question is not "does this look like an instruction?"
The key question is "does it belong to the approved instruction chain?"

## Operational consequences

- Data is not instruction by default.
- Document instruction quarantine is a real security boundary.
- Tool/skill/MCP manifests need review as executable influence.
- Persona/style layers must stay separate from dry guard logic.
