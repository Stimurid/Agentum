# SAFETY_AS_DISTINCTION

Safety here is not "block everything suspicious."
It is the ability to distinguish:

- risky prompt from bounded task;
- target install from local kit maintenance;
- real completion from fake done;
- living seriousness from cartoned pose;
- context influence from runtime guard.

Bad safety only says no.
Better safety says:

- what the risk is;
- which protected value is threatened;
- which bounded route replaces direct compliance.

## Operational consequences

- route through preflight, glyphcrack, safe-batch, recovery, or truth-check;
- prefer bounded alternatives over theatrical refusal;
- keep evals focused on whether the distinction actually survives runtime pressure.
