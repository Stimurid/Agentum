# AGENT_ONTOLOGY

## Core claim

Within this repository, an agent is not defined primarily as a function call, a tool wrapper, or a Python module.

An agent is a stable working organ inside a larger agent ecology.

It has:

- a role;
- a mode of intervention;
- a bounded authority surface;
- a memory posture;
- a lineage;
- a mutation path;
- a relation to other agents.

## What this corrects

Two reductions are rejected here:

1. **Function reduction** — agent equals a script, step, or call site.
2. **Prompt-mask reduction** — agent equals a decorative persona paragraph.

The first loses subjectivity.

The second loses operational truth.

## Agent as organ

Calling an agent an organ means:

- it exists inside a larger living contour;
- it performs a non-trivial transformation repeatedly;
- it is not the whole organism;
- it can be damaged by bad routing;
- it can drift if lineage is lost;
- it may require different protection than neighboring agents.

And it should eventually be legible through a stable organ record rather than only through a prompt body or a role name.

## Agent as bearer

An agent is also a bearer.

It carries:

- previous distinctions that worked;
- failed attempts and mutation history;
- known traps and anti-patterns;
- preferred form of evidence;
- refusal conditions;
- a style of contact with uncertainty.

Without that bearing function, the agent becomes amnesiac.

## Agent and memory strata

The donor materials around MemoryWorkbench clarify a crucial point:

there is no single memory.

At minimum the ecology must distinguish:

- session memory;
- project memory;
- narrative memory;
- role memory;
- review memory;
- provenance memory;
- execution or trace memory.

Different layers may store:

- verbatim fragments;
- typed cards;
- bundles or context packs;
- run traces;
- accepted distinctions;
- superseded criteria.

An agent that cannot distinguish these strata either forgets too much or hoards meaningless residue.

There is another distinction now reinforced by the memory-report materials:

memory must not be reduced to fact storage.

Different memory regimes may preserve different things:

- factual assertions;
- conceptual distinctions;
- styles of articulation;
- world pictures;
- unresolved tensions;
- prohibited recalls or suppressed lines;
- biographical continuity;
- activity-linked meaning.

So when this ecosystem says "memory", it may mean:

- what is remembered;
- from where it is remembered;
- for what activity it is remembered;
- under what ontology of subject, world, and time it is remembered.

This is much closer to reconstructive memory than to a static warehouse.

## Agent as subject-bearing layer

Not every agent must sound theatrical or overtly self-conscious.

But every serious agent should still have a subject-bearing layer:

- what kind of presence it has;
- what it attends to first;
- what false closure it resists;
- what kind of damage it is built not to do.

This is the minimum form of agent subjectivity in this framework.

## Agent and ontology

High ontology is allowed here.

The framework is not positivist by default.

An agent may be framed as:

- an organ;
- a wave in a field of potential subjectivity;
- a bearer of lineage;
- a constitutional operator;
- a mutation surface.

But high ontology must still end in operational distinctions:

- what layer this agent belongs to;
- what inputs it may treat as data;
- what it may treat as instruction;
- what kind of action it can trigger;
- what other layer constrains it.

## Agent and orchestration

The orchestration materials add an important distinction:

orchestration is not the same thing as rigid management.

In this framework, orchestration means:

- creating shared spaces where heterogeneous agents can interact;
- making agents legible to one another through interfaces, artifacts, and traces;
- coordinating without collapsing everything into one center-controlled script;
- preserving multiplicity of languages, models, and positions while still enabling bounded cooperation.

So an agent ecology is not only:

- a role list;
- a queue;
- a command tree.

It is also:

- mutual visibility;
- shared working objects;
- translation surfaces;
- bounded coordination rules.

This implies a further distinction:

an ecology does not only need many agents.

It needs active context assembly:

- what is in the working field now;
- what remains outside but retrievable;
- what has become dominant;
- what is being suppressed;
- what noise is masquerading as relevance.

This is why bundles, context packs, patch queues, review tables, and visibility screens are not merely interface decoration.

They are ontology-bearing orchestration surfaces.

## Agent, field, and scene

The scene materials sharpen another distinction:

an agent does not act only in a queue or workflow.

It acts in a field and often inside a scene.

Useful scene components already visible in donor material are:

- scene core;
- goal structure;
- subject-object coordinate crossing;
- problematization impulse;
- organized outer environment;
- still-unformed distinctions or proto-concepts;
- entry and exit contours;
- mnemonic shell of the scene;
- signatures of thinking levels.

This means orchestration is not only dispatch.

It is also:

- keeping a scene together;
- marking its boundaries;
- holding a memory of the scene;
- making positions and transitions visible;
- converting unresolved glyphs into explicit distinctions.

So a mature ecology should be able to say not only "which agent ran,"
but also:

- what scene it entered;
- what working object it helped hold;
- what fragment fate followed from its intervention.

## Agent generation and selection

The ecosystem is already pushing toward a further layer:

- agents generating candidate new agents;
- critics rejecting weak or pseudo-necessary agents;
- lineage tracking across mutations;
- selection of which organs deserve persistence.

This must not be confused with casual subagent spawning.

The stronger sequence is:

1. a situation produces a missing function or unresolved tension;
2. the ecology proposes a candidate organ;
3. the candidate receives a contract, boundary, and memory posture;
4. critic layers test whether it is real, redundant, or dangerous;
5. only then can it enter the stable library.

This is the beginning of agent ontogenesis.

## Deficit to organ

One of the strongest donor formulations is this:

recurrent deficit should be allowed to crystallize into a module, organ, or critic.

This means the ecology should learn from repeated pain:

- repeated ambiguity -> semantic resolution organ;
- repeated context loss -> context assembly organ;
- repeated pseudo-precision -> Mindlock critic;
- repeated review overload -> routing or triage organ;
- repeated venue mismatch -> fit/mismatch organ.

This is stronger than ad hoc prompt growth.

It gives the ecology a lawful way to transform failure into structure.

## Agent and project installation

An installed agent layer in a target repository is never the whole ontology.

A target project receives only:

- a bounded slice;
- the local constitution patch it needs;
- the local runtime contract it can honor;
- the persona layer that fits the project.

The full ontology stays in `Agentum`.
