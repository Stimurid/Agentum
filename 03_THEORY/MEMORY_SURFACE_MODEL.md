# MEMORY_SURFACE_MODEL

## Memory surface

Memory surface is every place remembered or recovered material can steer behavior:

- chat carryover;
- summaries;
- traces;
- notes;
- prior prompts;
- echo signatures;
- recovered files;
- old rules quoted back as if still current.

## Main risks

- stale memory execution;
- confabulated recall treated as evidence;
- memory poisoning through docs or prior outputs;
- past success becoming policy.

## Operational consequences

- Label material as recalled, anchored, inferred, or verified.
- Require provenance and scope before memory authorizes action.
- Use recovery to close the current evidence gap only.
- Stop retrieval when one bounded next step is supported.
