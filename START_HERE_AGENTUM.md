# START_HERE_AGENTUM

If you are a fresh Codex or Claude session entering this repository, read this file first.

## What this project is

`Agentum` is the master repository of a unified agent framework.

It is not just a protected coding-agent kit.

It is also:

- an ontology library for agents as organs, roles, and lineage-bearers;
- a subject-ecology framework;
- a distribution point for runtime guard layers and upper agent layers;
- a registry of which projects use which agent layers.

## What we are doing here

We are building one vertically integrated framework with two coupled branches:

1. **runtime/governance**
   - risk vector
   - preflight
   - glyphcrack
   - safe-batch
   - recovery
   - project identity
   - budget/stdout discipline
   - evals

2. **ontology/subjectivity**
   - agent constitution
   - role ecology
   - lineage
   - mutation logic
   - persona morphology
   - ecosystem control plane

The framework must not collapse into either:

- sterile function-only agents;
- decorative high-ontology cosplay without operational discipline.

## Recursive rule

This repository is a private project, but it is also its own first install target in a recursive sense.

Its own root files must already be "ours", not generic defaults.

That means:

- use Agentum-grade `AGENTS.md`;
- use Agentum-grade `CLAUDE.md`;
- use Agentum-grade `README.md`;
- maintain ecosystem registries and subject-ecology layer as real product parts;
- prefer replacing standard/project-boilerplate root docs with framework-quality docs.

## What not to do

- do not treat the whole archive as one runtime prompt;
- do not mix persona humor into dry guard files;
- do not confuse ontology with deploy/runtime product behavior;
- do not sync target projects without reviewed profile logic;
- do not rewrite project content when only the agent-evolution contour should change.
