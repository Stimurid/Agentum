# AGENTS.md — Agentum

Agentum is the master repository for two coupled branches:

1. runtime protection / governance / install discipline;
2. agent ontology / subjectivity / ecosystem evolution.

This repository is infrastructure, but it is also self-hosting: its own root agent files must follow the same standards that it exports to other projects.

Critical rules first:

- Treat `Agentum` as infrastructure, not as a target application repo.
- Work only inside this repository unless the task explicitly shifts to a reviewed target-project install.
- Preserve corpus provenance, registries, pattern lineage, failure modes, install profiles, eval scenarios, and handoff prompts.
- Do not collapse the research archive or ontology corpus into one runtime blob.
- Do not rewrite registries without a concrete consistency reason.
- Large bounded batches are allowed here; forced microtasking is not required.
- “Stop microtasking” never means rewrite-everything.
- UI / UX / operator-surface work must not be improvised from vibes; read the interface guard before changing interfaces.
- Validate before commit.
- Do not claim done without checks, diffs, or honest status.
- Russian handoff prompts are the default. Technical file names and command names may remain in English.
- Persona humor fragments must stay separate from dry guard/runtime enforcement files.

Recursive rule:

- This project must use its own updated standards on itself.
- Root `AGENTS.md`, root `CLAUDE.md`, `README.md`, install docs, profiles, and ecosystem registries are not decorative. They are part of the working product.
- When improving Agentum, prefer replacing generic root files with Agentum-grade files instead of leaving placeholders or standard boilerplate.

Working rules:

- Name the working layer before editing: root docs, corpus, theory, v0 kit, subject ecology, persona fragments, profiles, install docs, ecosystem, evals, handoff prompts.
- Prefer compact, operational docs over archival bloat.
- Keep subject layer distinct from runtime layer.
- Keep ecosystem sync distinct from target-project content work.
- Commit only after local checks pass and staged content is clean.
