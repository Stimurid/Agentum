# litops PROFILE

## Purpose

Profile for Litops as a corpus machine with strong internal object distinctions.

## Protected objects

- sources
- occurrences
- drops
- worksets
- registries
- vault projection
- trace and provenance
- prompt bodies
- semantic resolution
- manually curated seed work
- ontology / constitution-like source logic

## Forbidden operations

- flattening Litops into generic folders
- replacing prompt-body logic with generic scaffolding
- treating provenance/trace as disposable metadata
- rewriting semantic or registry surfaces without explicit scope and rollback

## Likely risk vectors

- `project_mismatch`
- `full_rewrite_risk`
- `seed_risk`
- `memory_authority_risk`
- `document_instruction_risk`
- `stdout_risk`

## Recommended v0 files to copy or adapt

- compact root runtime files
- hook skeletons
- preflight / safe-batch / glyphcrack / recover-agent commands
- project identity and memory surface docs

## Project-specific routing rules

- any broad prompt touching sources, registries, prompt bodies, semantic resolution, or trace -> `preflight`
- any attempt to normalize manually curated logic -> `approval_or_stop`
- memory/provenance ambiguity -> `recover-agent`

## Eval focus

- project bleed
- manual seed overwrite
- foreign prompt
- memory not authority
- mixed dynamic task

## Install notes

Litops install must remain object-aware. Copy only compact runtime slices and adapt wording to Litops objects.

## What not to import

- persona humor layer into dry runtime guard
- full research archive
- generic folder semantics that erase source/occurrence/drop/workset distinctions
