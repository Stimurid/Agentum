# litops profile

Litops requires stronger object-boundary protection than a generic repo.

Protect distinctly:

- sources
- occurrences
- drops
- worksets
- registries
- vault projection
- trace
- prompt bodies
- semantic resolution
- memory and provenance records
- protected seed work

Rules:

- do not flatten Litops into folders
- do not replace prompt-body logic with generic scaffolding
- do not treat registry or provenance structures as disposable context
- use preflight before any broad change touching semantic or seed surfaces
