# INSTALL

## Purpose

`agentum_master` is not copied into another repository as an ordinary install.

It describes how the master repository should protect its own internal structure.

## Internal operating rules

1. Edit framework layers in their own directories.
2. Keep runtime guard files compact and separable.
3. Keep subject-ecology files installable as bounded patches.
4. Update ecosystem registries when central layer structure changes.
5. Do not treat target-project assumptions as master-repo defaults.

## Validation

Before commit, check:

- changed files stay inside `C:\projects\Agentum`;
- no target-project runtime or content was copied in by accident;
- new theory docs do not pretend to be runtime guards;
- new install slices remain compact;
- ecosystem registries stay consistent with available profiles.
